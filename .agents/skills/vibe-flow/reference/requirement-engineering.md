# 结构化需求工程

## 核心能力
- BRD商业需求文档
- 模块化PRD撰写
- 业务流程梳理
- 角色权限定义
- 正常流程+异常流程+边界场景全覆盖
- 需求拆分

## Vibe结合
需求完全书面化、无歧义，AI可精准理解业务，不会写出逻辑错乱代码。

## 核心产出
- 主PRD文档
- 各模块子需求文档
- 业务流程图
- 角色权限矩阵

## 使用指南

### 1. BRD商业需求
- 商业目标
- 市场定位
- 竞品分析
- 成功指标

### 2. PRD结构化
- 功能模块划分
- 用户故事编写
- 验收标准定义
- 优先级排序

### 3. 流程全覆盖
- 正常流程（Happy Path）
- 异常流程（错误处理）
- 边界场景（极限值、空值、并发）

### 4. 需求拆分原则
- 独立性（Independent）
- 可协商（Negotiable）
- 有价值（Valuable）
- 可估算（Estimable）
- 小型化（Small）
- 可测试（Testable）

## AI提问模板
```
基于以下产品描述，请帮我：
1. 撰写完整的PRD文档
2. 梳理所有业务流程（正常+异常+边界）
3. 定义角色权限矩阵
4. 拆分需求为可开发的用户故事
```

# 商业立项 & 项目范围定义

## 核心能力
- 用户画像分析
- 痛点定位
- MVP边界划定
- 做什么/绝不做什么
- 盈利模式设计
- 项目生命周期规划
- 风险预判

## Vibe结合
前期锁死范围，开发中绝不乱加需求，保证心流不被打断。

## 核心产出
- 项目立项书
- 范围边界清单
- MVP规划文档

## 使用指南

### 1. 用户画像分析
明确目标用户群体，包括：
- 用户特征（年龄、职业、技术背景）
- 核心痛点
- 使用场景
- 付费意愿

### 2. MVP边界划定
- 核心功能（必须有）
- 次要功能（可以有）
- 未来功能（以后做）
- 绝不做的功能

### 3. 盈利模式设计
- 免费增值模式
- 订阅制
- 一次性付费
- 广告模式

### 4. 风险预判
- 技术风险
- 市场风险
- 时间风险
- 资金风险

## AI提问模板
```
我正在开发一个[产品类型]，目标用户是[用户群体]。
请帮我：
1. 分析核心痛点和使用场景
2. 划定MVP功能边界（必须有/可以有/以后做/绝不做）
3. 设计合适的盈利模式
4. 预判可能的风险点
```

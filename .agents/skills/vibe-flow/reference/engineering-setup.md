## 使用指南

### 1. Git分支规范（单人推荐轻量）
- main分支（生产）
- feature短分支（功能，短周期合并）
- hotfix分支（线上修复）
- tag/release notes（版本发布记录）
- 可选：项目复杂后再引入 develop/release 分支

### 2. 代码规范
- ESLint配置
- Prettier配置
- Commit规范
- 代码审查流程

### 3. 多环境隔离
- 开发环境
- 测试环境
- 预发布环境
- 生产环境

### 4. 全局拦截器
- 请求拦截
- 响应拦截
- 错误拦截
- 日志拦截

### 5. 统一异常处理
- 异常分类
- 异常捕获
- 异常日志
- 异常恢复

### 6. 工具类库
- 日期处理
- 字符串处理
- 数据校验
- 加密解密

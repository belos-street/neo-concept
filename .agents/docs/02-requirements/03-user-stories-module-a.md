## US-A1 — 首次启动获取课程索引

> **作为** 学习者
> **我希望** 首次打开 App 时自动获取课程索引
> **以便** 我能看到所有可学的课程列表

### 验收标准

- [ ] AC-A1.1：首次启动时，从内置的 GitHub Pages URL 拉取 `manifest.json`
- [ ] AC-A1.2：若拉取成功，解析并展示 Book → Unit → Lesson 树形列表
- [ ] AC-A1.3：若拉取失败（无网络/URL 错误），显示错误提示并允许手动重试
- [ ] AC-A1.4：manifest 本地缓存，后续启动使用缓存，不重复拉取（除非手动刷新）
- [ ] AC-A1.5：首次启动时，从 APK assets 复制 ECDICT 数据库文件（.db）到 SQLite 目录（~5 MB），复制期间显示「词典准备中...」，完成后所有单词 tooltip 功能可用

### 异常流程

- **无网络**：显示「无法连接到课程仓库，请检查网络后重试」，提供「重试」按钮
- **manifest 解析失败**：显示「课程索引格式错误，请联系维护者」
- **ECDICT 数据库初始化失败**：不阻塞 App 使用；单词 click 事件依然绑定，tooltip 显示「词典暂不可用」

### 空状态

首次启动 manifest 未拉取成功时，课程列表区域显示「课程列表加载中，请确保已连接网络」，提供「重试」按钮。

## US-A2 — 按需下载单课内容

> **作为** 学习者
> **我希望** 点击某课时才下载该课内容
> **以便** 不用一次性下载所有课程，节省存储空间

### 验收标准

- [ ] AC-A2.1：点击课程列表中的某课 → 显示下载进度 → 下载完成后自动进入 Step 1
- [ ] AC-A2.2：下载完成后的课程 JSON 本地持久化，下次点击直接进入学习（不再下载）
- [ ] AC-A2.3：显示已下载/未下载状态标识（如 🟢 已下载 / ⬜ 未下载）

### 异常流程

- **下载中断（网络断开/App 退出）**：下次点击该课时从头重新下载
- **下载内容 Hash 校验失败**：提示「内容校验失败，可能文件已损坏，请重试」，自动重新下载
- **存储空间不足**：提示「存储空间不足，请释放空间后重试」
- **JSON 解析失败**：提示「课程内容格式错误」，不上屏

### 边界场景

- 同时点击多课下载 → 串行排队下载，不并发；课程列表中显示排队序号（如「排队中 2/3」）
- 下载过程中切换页面 → 下载继续后台执行
- 正在排队/下载的课程可点击「取消」，取消后该课恢复为「未下载」状态

## US-A3 — 检查课程更新

> **作为** 学习者
> **我希望** 手动检查课程是否有更新
> **以便** 获取新增或修正的课程内容

### 验收标准

- [ ] AC-A3.1：「检查更新」按钮 → 重新拉取 manifest → 与本地缓存的 manifest 对比
- [ ] AC-A3.2：对比每课的 SHA256 Hash，标记有变更的课程
- [ ] AC-A3.3：有更新的课程显示「更新可用」标识，用户可选择增量更新
- [ ] AC-A3.4：课程内容更新（Hash 变化）后，该课已有学习进度自动重置为 Step 1，已完成 Step 的回看数据一并清除；更新前弹出提示「本课内容有更新，已有学习进度将被重置」

### 异常流程

- **远程 manifest 版本低于本地**：不降级，提示「已是最新版本」
- **检查更新时网络不可用**：提示「网络不可用，无法检查更新」

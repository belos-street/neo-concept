## 学习主流程（Happy Path）

```
用户打开 App
  │
  ▼
显示课程树（Book → Unit → Lesson）
  │
  ▼
点击某课
  ├─ 已下载 → 直接进入
  └─ 未下载 → 下载 JSON + 图片 → 进入
  │
  ▼
进入 Step 1（课文预习）
  │ 读课文 + 听朗读 + 看语法点
  ▼
Step 2（填词练习）
  │ 6-8 题
  ▼
Step 3（词汇记忆）
  │ 闪卡 → 拼写 → 匹配
  ▼
Step 4（听力练习）
  │ 4-5 题选择题
  ▼
Step 5（阅读理解）
  │ 5 题选择题
  ▼
Step 6（口语练习）
  │ 3-5 句跟读
  ▼
一课完成 🎉
```

---

## 碎片学习流程（中断恢复）

```
08:15  Step 1 完成 → 进度写入 MMKV (currentStep=2)
       → 用户退出

12:30  用户重新打开该课
       → 显示过渡界面：「继续学习（Step 3）」+「回顾已完成内容」
       → 用户选择「继续学习」→ 跳转 Step 3
       → Step 3 完成 → currentStep=4
       → 退出

21:00  重新打开 → 过渡界面 → 跳转 Step 4
       → Step 4-6 完成
       → lessonCompleted=true
```

## 内容下载流程

```
App 启动
  │
  ├─ 首次启动
  │   ├─ fetch manifest.json（从配置的 Git 仓库 URL）
  │   ├─ 成功 → 缓存到本地 → 解析显示课程列表
  │   └─ 失败 → 显示错误 + 重试按钮
  │
  ├─ 非首次启动
  │   └─ 使用本地缓存的 manifest
  │
  ▼
用户点击某课
  ├─ 本地已有该课 JSON → 直接打开
  └─ 本地没有 → 下载
      ├─ 下载 JSON + 图片
      ├─ Hash 校验 → 缓存到本地 → 打开
      └─ 失败 → 错误提示 + 重试
```

---

## 异常流程总览

| 场景 | 处理方式 |
|------|---------|
| 首次启动无网络 | 显示错误页 + 「重试」按钮；不阻塞进入 App（可查看已缓存内容） |
| 下载课程 JSON 失败 | 错误提示，该课保持未下载状态 |
| Hash 校验失败 | 自动重新下载 |
| 存储空间满 | 提示清空间 |
| TTS/ASR 模型未就绪 | 首次启动从 APK assets 提取中，显示加载状态，完成后自动启用 |
| 录音权限未授予 | 引导到系统设置 |
| ECDICT 数据库初始化失败 | 单词 tooltip 显示「词典加载中」 |
| MMKV 读写失败 | 静默降级，进度不持久化但学习可继续 |
| JSON 内容格式错误 | 不上屏，提示「内容格式错误」 |

# Neo Concept — 任务拆解与验证计划

> 状态：待用户确认
> 目标：将 MVP 开发拆分为可执行、可验证的最小任务单元，明确每步完成标准与手动验证方法。

---

## 开发原则

- 按依赖顺序推进，前一个任务不通过不进入下一个。
- 每个任务必须包含：实现内容、完成标准（DoD）、手动验证方法、验证点。
- 优先打通「内容读取 → 首页 → 书详情 → 课前导读 → 课文学习 → 完成页」主链路，再补全练习步骤与语音引擎。

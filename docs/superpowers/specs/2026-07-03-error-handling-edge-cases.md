# Neo Concept — 错误处理与边界情况设计方案

> 状态：已确认
> 目标：明确各异常场景下的降级策略与用户提示方式，保证离线优先体验不中断。

---

## 1. 设计原则

- **不闪退**：任何内容或资源异常都不应导致 App 崩溃，至少回到可继续操作的状态。
- **离线优先**：网络异常视为常态，优先使用本地占位或降级内容。
- **最小打扰**：能用静默降级解决的，不弹窗；必须让用户知道的，用 Toast 或内联提示。
- **进度不丢**：学习过程中任何中断，已完成的步骤和当前位置都应被保存。

---

## 2. 已确认决策

### 2.1 网络图片加载失败（课程 Banner）

- **策略**：失败后直接显示本地默认占位图，不显示「网络错误」文字提示。
- **不显示重试按钮**：离线使用场景下，占位图即为默认状态。
- **占位图**：使用课程标题文字 + 纯色/图案背景。

### 2.2 TTS 播放失败

- **策略**：不引入系统 TTS 降级。
- **提示**：弹出 Toast「音频加载失败，请重新进入 App」。
- **后续**：提示后关闭 App，由用户下次重新启动恢复。

### 2.3 ASR 不可用

- **策略**：同时显示「跳过口语」和「重试」两个按钮。
- **不区分原因**：ASR 初始化失败、模型缺失、设备性能不足统一按此处理。
- **不记录错误历史**：不单独标记「ASR 失败导致的跳过」。

### 2.4 JSON 损坏/缺失

- **策略**：首次启动时建立内容索引并校验完整性。
- **损坏课程**：不显示在课程列表中，同时弹出 Toast「部分课程暂不可用」。
- **其余课程**：正常加载，保证 App 可用性。

### 2.5 学习中途退出 / 接电话 / 切换应用

- **策略**：每次步骤切换或完成时立即写入进度。
- **返回行为**：来电或切换应用后返回学习页，直接停留在原步骤，不需要重新加载或提示。
- **启动恢复**：启动 App 时不自动跳转学习页，仍通过首页「继续学习」卡片恢复（此前已确认）。

### 2.6 无麦克风权限时口语练习降级

- **首次拒绝**：再次进入口语练习时继续弹权限请求。
- **永久拒绝**：显示 Toast「需要麦克风权限进行口语练习」，并提供「去设置」入口。
- **跳过口语**：始终保留「跳过口语」按钮；跳过后完成页显示「口语待补」标记（此前已确认）。

---

## 3. 全局异常提示规范

| 异常类型 | 提示方式 | 文案示例 |
|----------|----------|----------|
| 网络图片失败 | 静默降级（占位图） | 无 |
| TTS 失败 | Toast + 关闭 App | "音频加载失败，请重新进入 App" |
| ASR 不可用 | 内联提示 + 两个按钮 | "口语练习暂不可用" + [跳过口语] [重试] |
| JSON 损坏 | Toast | "部分课程暂不可用" |
| 权限拒绝（可再次请求） | 系统权限弹窗 | 系统默认文案 |
| 权限永久拒绝 | Toast + 去设置入口 | "需要麦克风权限进行口语练习" |
| 进度保存失败 | Toast | "进度保存失败，请重试" |

---

## 4. 实现要点

- **TTS 失败关闭 App**：在 Toast 显示后调用 `finishAffinity()` 或等效退出，不强制杀进程。
- **JSON 校验**：首次启动在后台线程解析 `manifest.json` → `book.json` → `lesson.json`，记录损坏的 `bookId`/`lessonId`，写入本地索引时排除。
- **进度写入时机**：每个 Step 完成、用户点击「下一步」、或 `onPause` 触发时，都执行一次 `LessonProgress` 持久化。
- **权限请求**：使用 Android `registerForActivityResult` 管理权限请求；永久拒绝时通过 `Intent` 跳转应用详情页。

---

## 5. 边界情况补充

- 如果全部课程 JSON 均损坏，首页书架显示为空，Toast 提示「课程加载失败」。
- 如果用户在学习页触发 TTS 失败，关闭 App 前仍先保存当前进度。
- 如果 ASR 在口语练习中途失败（已开始录音后），停止录音并显示「跳过/重试」选项。

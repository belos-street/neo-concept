# Neo Concept 移动端设计系统 — Swiss Minimalist（适配版）

> 适用范围：Android（Jetpack Compose）与 iOS（SwiftUI）
> 状态：已确认方向
> 基调：吸收 Swiss International 设计理念（客观、网格、字体即结构、高对比、大量留白、扁平），针对移动学习场景做适应性改造。

---

## 1. 设计哲学

- **内容优先**：界面退后，让课文、单词、练习本身成为焦点。
- **客观清晰**：避免装饰性元素，每个视觉决策都服务于信息层级。
- **网格为骨**：以 4pt 网格为基准组织间距与对齐，但不强求可见纹理。
- **字体即界面**：通过字号、字重、字间距建立层级，而非依赖图标或色彩堆叠。
- **扁平与秩序**：不使用阴影和渐变制造深度，深度来自边框、背景色块和留白。
- **移动友好**：保留瑞士风的高对比与秩序感，同时通过小圆角、语义色和舒适字号适配单手操作。

## 2. 设计 Token

### 2.1 颜色

#### 主色板（严格克制）

| Token | 浅色模式 | 深色模式 | 用途 |
|-------|----------|----------|------|
| `background` | `#FFFFFF` | `#0A0A0A` | 页面主背景 |
| `foreground` | `#000000` | `#F5F5F5` | 主文字、图标、边框 |
| `muted` | `#F2F2F2` | `#1A1A1A` | 次级背景、卡片、分隔 |
| `accent` | `#FF3000` | `#FF4D1A` | 主强调色：CTA、当前步骤、重点标记 |
| `border` | `#000000` | `rgba(245,245,245,0.15)` | 结构边框 |

#### 语义状态色

| Token | 浅色模式 | 深色模式 | 用途 |
|-------|----------|----------|------|
| `success` | `#006E2C` | `#3DCF6A` | 回答正确、已完成 |
| `error` | `#B3261E` | `#FF775C` | 回答错误、删除、强警告 |
| `warning` | `#B25E00` | `#FFB74D` | 提示、需留意 |
| `info` | `#005FA8` | `#6EC6FF` | 辅助说明 |
| `locked` | `#9E9E9E` | `#666666` | 未解锁、禁用 |
| `completed` | `#000000` | `#FFFFFF` | 已完成的步骤/课程（主色即可表达） |

> 原则：强调色（红）只用于主 CTA、当前状态和章节编号；状态色仅用于反馈，不当作装饰。

### 2.2 字体

- **Android**：`Inter` 为主字体，系统无衬线字体作为 fallback。
- **iOS**：优先使用 `SF Pro`；如需跨平台一致，可使用 `Inter`。
- **字重**：
  - `Black 900` / `Bold 700`：大标题、章节编号、按钮。
  - `Medium 500`：小标题、标签。
  - `Regular 400`：正文、例句、释义。
- **大小比例（移动端基准）**：
  - `display`：40sp / 48sp（仅用于首页大标题）。
  - `headline`：28sp / 32sp（页面标题、书名）。
  - `title`：20sp / 24sp（课程标题、步骤名）。
  - `body`：16sp / 18sp（课文、例句、说明）。
  - `label`：12sp / 14sp（标签、小字、音标辅助）。
- **样式规范**：
  - 标题与按钮标签使用**大写**（英文）或**加粗**（中文），增强秩序感。
  - 正文严格左对齐，避免居中对齐大段文字。
  - 行高：标题 1.1–1.2，正文 1.5–1.6。

### 2.3 间距

以 **4dp** 为基数：

| Token | 值 | 用途 |
|-------|-----|------|
| `space-1` | 4dp | 图标与文字间距、紧凑内联 |
| `space-2` | 8dp | 行内元素间距 |
| `space-3` | 12dp | 小卡片内边距 |
| `space-4` | 16dp | 标准页面左右边距、组件间距 |
| `space-5` | 24dp | 区块间距、大卡片内边距 |
| `space-6` | 32dp | 章节分隔 |
| `space-7` | 48dp | 首页 Hero 区域留白 |

> 页面左右安全边距固定为 `16dp`（小屏）或 `24dp`（大屏平板）。

### 2.4 圆角与边框

- **圆角**：
  - 结构性容器：`0dp`（严格矩形）。
  - 按钮、输入框、卡片、Modal：`4dp–8dp`，提升移动触控舒适度。
  - 圆形元素（步骤小圆点、头像、播放按钮）：`50%`。
- **边框**：
  - 默认：`1dp` 黑色（浅色）/ 低透明度白（深色）。
  - 强调分隔：`2dp` 黑色。
  - 不使用阴影，边框即结构。

### 2.5 动效

- **时长**：150ms–200ms，快速、机械、无弹性。
- **缓动**：`ease-out` 或线性。
- **交互反馈**：
  - 按钮按下：背景色反转（黑 ↔ 白）或切换为强调色。
  - 卡片按下：轻微下沉 `1dp` 或背景变为 `muted`。
  - 图标状态变化：旋转 90°（展开/收起）、缩放 1.05（数字强调）。
- **减弱动效**：所有动画必须响应系统 `prefers-reduced-motion` / `reduceMotion` 设置。

## 3. 布局原则

- **4pt 网格**：所有尺寸、间距、内边距对齐 4 的倍数。
- **安全区**：尊重系统状态栏、导航栏、刘海/挖孔、IME 键盘区域。
- **Edge-to-edge**：内容延伸至屏幕边缘，关键操作避开手势区域。
- **学习页秩序**：课文页、练习页以单列为主，保证可读性；不对称布局仅用于首页/营销区。
- **可见结构**：用边框和背景色块划分区域，而非阴影。

## 4. 组件规范（概要）

### 4.1 按钮

- **主按钮**：黑色背景 + 白色文字，大写/加粗，圆角 4dp，高度 48dp–56dp。
- **次按钮**：白色背景 + 黑色边框 + 黑色文字。
- **强调按钮**：强调色背景 + 白色文字，仅用于最重要的单一操作。
- **禁用状态**：背景 `locked` 色，文字 `background` 色。
- **按下反馈**：颜色反转或整体切换为强调色，200ms 内完成。

### 4.2 卡片

- 白色或 `muted` 背景，黑色边框，圆角 8dp。
- 内边距：`16dp` 或 `24dp`。
- 选中/悬停：边框变为强调色或背景色反转。

### 4.3 输入框

- 下划线样式（`border-b-2`）或带边框矩形框。
- 聚焦时边框变为强调色，无发光阴影。
- 错误状态边框为 `error` 色，下方显示简短错误提示。

### 4.4 步骤指示器

- 6 个小圆点，直径 8dp，间距 8dp。
- 当前步骤：强调色实心。
- 已完成：主色（黑/白）实心。
- 未解锁：`locked` 色空心或实心。

### 4.5 查词 Modal

- 居中或底部 Sheet，白色/深色背景，黑色/浅色边框。
- 内容：单词（大字号、可点击发音）、音标、中文释义、例句。
- 关闭按钮位于右上角，使用简洁几何图标。

### 4.6 标签与徽章

- 矩形或微圆角小标签，黑色边框。
- 状态徽章：已完成（黑底白字）、进行中（强调色底白字）、未开始（白底黑字）。

## 5. 核心页面组件布局详述

以下布局以 360dp 宽手机为基准，左右安全边距 `16dp`。

### 5.1 首页继续学习卡片

```
┌────────────────────────────────────┐
│  ┌────────────────────────────┐    │
│  │ CONTINUE LEARNING          │    │
│  │ ─────────────────────────  │    │
│  │ Book 1 · Lesson 03         │    │
│  │ A Private Conversation     │    │
│  │                            │    │
│  │  ● ○ ○ ○ ○ ○   Step 2 of 6│    │

│  │ [     CONTINUE LEARNING    ]│   │
│  └────────────────────────────┘    │
└────────────────────────────────────┘
```

- **位置**：首页顶部，状态栏下方，`space-5` 边距。
- **容器**：白色/深色卡片，黑色边框，圆角 `8dp`，内边距 `space-5`。
- **标题**：`label` 大写英文「CONTINUE LEARNING」，字重 Medium。
- **书名与课名**：`body` 字号，Muted 色。
- **课程标题**：`title` 字号，前景色，单行截断。
- **步骤指示器**：6 个小圆点，`8dp`，当前步骤为强调色实心，其余为 `locked` 色。
- **按钮**：主按钮「CONTINUE LEARNING」，黑色背景，全宽，高度 `48dp`。

### 5.2 首页书卡片

```

│  01        FIRST THINGS FIRST   →  │
│  ───────────────────────────────── │
│  144 课                已完成 72   │
```

- **容器**：全宽卡片，`muted` 背景或白色，黑色边框，`8dp` 圆角，内边距 `space-5`。
- **左侧大编号**：`display` 字号（40sp），占据卡片左上角，前景色或强调色。
- **书名**：`headline` 字号，大写英文，右对齐或与大编号同行靠右。
- **分隔线**：`1dp` 黑色，贯穿卡片，`space-3` 上下边距。
- **课程数量**：`body` 字号，左侧。
- **进度**：右侧显示「已完成 N / 总课数」，下方为细进度条，高度 `4dp`。
- **箭头**：右上角 `24×24dp`，进行中用强调色，已完成用 `success` 色，未开始用 `border` 色。
- **状态**：
  - 进行中：边框 `2dp`，箭头强调色。
  - 已完成：背景反转为黑色，文字白色，对勾图标。
  - 未开始：默认样式。

## 6. 语义模式

### 6.1 学习反馈

- **正确**：使用 `success` 色，配合对勾图标；背景可轻微着色。
- **错误**：使用 `error` 色，配合叉号图标；显示正确答案或提示。
- **锁定**：使用 `locked` 色，降低透明度，显示锁图标。

### 6.2 课程状态

- **未开始**：白底黑字卡片。
- **进行中**：步骤小圆点或标签使用强调色。
- **已完成**：标题旁加对勾，卡片边框变粗或背景变 `muted`。
- **口语待补**：用 `warning` 色小标签提示。

---

## 7. 深色模式

- 背景 `#0A0A0A`，前景 `#F5F5F5`。
- `muted` 用于卡片和输入框背景。
- 边框使用低透明度白色，避免纯黑背景上的生硬白线。
- 强调色在深色下略微提亮（`#FF4D1A`），保持冲击力。
- 图片/插画在深色模式下无需反转，但 banner 占位背景应适配。

---

## 8. 平台实现提示

- **Android / Jetpack Compose**：
  - 使用 `MaterialTheme` 或完全自定义 `Theme` 注入 Color/Shape/Typography Token。
  - 禁止直接使用 Material3 默认样式的按钮/卡片；所有组件按本系统封装。
  - 使用 `CompositionLocalProvider` 或自定义 design system 对象管理 Token。
- **iOS / SwiftUI**：
  - 定义 `Color` 和 `Font` extension 存放 Token。
  - 使用自定义 `ViewModifier` 统一按钮、卡片样式。

---

## 9. 可访问性

- 正文与背景对比度至少 4.5:1；大标题与强调元素至少 3:1。
- 所有可点击区域不小于 44×44dp。
- 支持系统字体大小缩放，布局不被截断。
- 动画可被系统减弱动效设置关闭。

---

## 10. 不采用的元素

以下原版 Swiss 规范中的元素，因不适合移动学习场景或实现成本过高，本项目不予采用：

- CSS 纹理图案（grid、dot、diagonal、noise）。
- 0px 圆角用于所有元素（改为 4–8dp 的小圆角）。
- 纯黑白红三色板（增加语义状态色）。
- Web 端超大标题比例（移动端按比例缩小，但仍保持层次）。
- 复杂的悬停动效（改为按压/触摸反馈）。
